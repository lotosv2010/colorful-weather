# 霁色天气 — 功能差距分析

> 对标 Apple Weather、彩云天气、墨迹天气、Windy 等主流天气应用
> 最后更新：2026-07-13

---

## 第一梯队：核心竞争力缺失

| 功能 | 说明 | 实现难度 | 和风 API |
|------|------|----------|----------|
| **天气雷达 / 降水地图** | 可视化降水位置和移动方向，Apple Weather 头号招牌功能。目前只有分钟级柱状图，没有地图图层 | 高 | 格点天气 / 地图服务 |
| ~~**多城市横划切换**~~ | ✅ 已实现：紧凑卡片左右滑动切换定位城市与收藏城市，圆点指示器 | — | — |
| ~~**多城市概览面板**~~ | ✅ 已实现：右上角城市列表按钮，全屏上滑面板展示各城市实时天气 / 最高低温，点击直接切换 | — | — |
| **自然语言天气摘要** | 每个数据卡片附一句解释（"比昨天低 3°，风寒体感更明显"），是 Apple Weather 核心体验 | 低（模板文案，无需 AI）| — |
| **点击 7 天预报展开当日逐小时** | 标配交互，目前 7 天列表只能跳到全局 hourly 页，无法查看指定日期 | 中 | — |

## 第二梯队：数据维度缺失

| 功能 | 说明 | 实现难度 | 和风 API |
|------|------|----------|----------|
| **逐小时湿度曲线** | hourly 页目前有温度、降水、风速三图，缺少湿度趋势图 | 低 | 已在逐小时接口中 |
| **具体降水量（mm）** | 分钟级和逐小时只有概率 / 强度分级，缺少 mm 数值 | 低 | 已在逐小时接口中 |
| **云量（Cloud Cover）** | Apple Weather 有专门云量卡片，影响光照、拍摄、太阳能发电预估 | 低 | 已在逐小时接口中 |
| **月出 / 月落时间** | 有月相展示，但缺少月亮升落时刻（astronomy 组件未实现） | 低 | 天文接口 |
| **风寒指数 / 暑热指数（Heat Index）** | 体感温度背后的量化解释，Apple Weather 有 | 中 | — |
| **能见度趋势图** | 实时能见度数值已有，缺逐小时趋势 | 低 | 已在逐小时接口中 |

## 第三梯队：中国本地化

| 功能 | 说明 | 实现难度 | 和风 API |
|------|------|----------|----------|
| **农历日期 / 二十四节气** | 7 天 / 30 天列表中显示，中国用户强需求，彩云天气 / 中国天气通标配 | 低（纯前端算法，无需 API）| — |
| **"一句话天气"文案** | 彩云天气标志性设计，几个字说清今天的天气主要特征 | 低（模板 + 规则）| — |
| **花粉 / 过敏预报** | Apple Weather iOS 16+ 新增，适合过敏体质用户 | 中 | 生活指数中有花粉相关 |
| **AQI 健康建议文案** | 空气质量页只显示数字等级，缺少"不建议户外运动"等行动指导 | 低（模板文案）| — |

## 第四梯队：平台能力

| 功能 | 说明 | 实现难度 |
|------|------|----------|
| **桌面快捷入口引导** | 引导用户固定小程序到桌面（`wx.addShortcut`），降低下次打开门槛 | 低 |
| **分享卡片带天气快照** | 分享路径已有，但如能生成含温度 / 天气图标的自定义预览图更吸引人 | 高（Canvas 生成图片）|

## 优先级建议

按投入产出比排序，建议优先补：

1. **多城市横划切换** — 纯交互层改动，ROI 最高，用户留存提升明显
2. **自然语言摘要文案** — 模板实现，0 API 成本，大幅提升"有温度感"
3. **点击 7 天预报展开当日逐小时** — 标准交互，用户强预期
4. **逐小时湿度图** — hourly 页补第四图表，数据已在接口中，改动最小
5. **农历 / 节气** — 面向中国用户，纯前端实现，无需新 API

天气雷达（降水地图）体验提升最大，但接入成本最高，适合作为长期目标。
